# Plan 001 - MVP

## Objective

Build a demo-ready Quote Delta Ledger prototype for one moving-services negotiation scenario.

## Outcome Map

- User outcome: compare vendor offers without losing track of fees, terms, and concessions.
- Product outcome: turn calls into an evidence-backed quote ledger.
- Demo outcome: show one verified before/after negotiation delta.
- Judge outcome: prove this is a negotiation system, not a generic call summary.
- Technical outcome: keep the app typed, tested, and verifiable with one command.

## Scope

- Monorepo product structure with clear docs, skills, app boundaries, and runbooks.
- Single-page React app in `apps/web`.
- Seeded job spec and three vendor transcript fixtures.
- Quote ledger with itemized fees and evidence links.
- Honesty guardrail for competing-bid leverage.
- Negotiated delta view.
- Ranked recommendation report.

## Non-Scope

- Real production telephony.
- User accounts.
- Multi-vertical configuration UI.
- Payment, booking, or contract execution.
- Storing private real customer recordings.

## Milestones

- [x] Repository constitution and product docs.
- [x] Monorepo structure with apps, packages, docs, runbooks, skills, and workflows.
- [x] Initial app shell and fixture-backed quote ledger.
- [x] Honesty guardrail unit tests.
- [x] E2E demo path.
- [x] Transcript span inspector.
- [x] Follow-up negotiation state transition.
- [x] Exportable report.
- [x] Optional Tavily verification slice.
- [x] Optional OpenAI structured extraction slice.
- [x] Real ElevenLabs browser voice session with AI disclosure and microphone input.
- [x] Live transcript, conversation ID, audio-proof retrieval, and explicit provider event log.
- [x] Strict live-call mode with no fixture fallback.

## Acceptance Criteria

- A user can see one structured moving job spec.
- The app displays three comparable vendor quotes.
- Every quote row exposes transcript evidence.
- At least one fee is marked vague or hidden.
- One follow-up negotiation shows a measurable price or term delta.
- Unsupported leverage is refused in domain logic.
- `./scripts/check.sh` passes.
- A real ElevenLabs call connects, speaks with the configured voice, and returns post-call proof.
- Live provider failures are shown with their stage and HTTP/provider message.

## Testing Strategy

- Unit-test quote total and delta calculations.
- Unit-test honesty guardrails.
- Smoke-test the core browser workflow.
- Keep fixture data deterministic for demo reliability.
