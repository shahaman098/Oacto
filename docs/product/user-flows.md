# User Flows

## Flow 1: Confirm Service Request

1. User enters or confirms the job spec.
2. System normalizes the request into fields that every vendor can answer.
3. User sees the confirmed spec before calls begin.

Expected outcome: every vendor is evaluated against the same request.

## Flow 2: Review Vendor Calls

1. System ingests call transcripts or fixture transcripts.
2. Quote assertions are extracted from each transcript.
3. User reviews each vendor card with price, terms, fees, and evidence.

Expected outcome: messy calls become comparable rows.

## Flow 3: Detect Negotiation Leverage

1. System compares all confirmed quote assertions.
2. Unsupported or vague claims are excluded from leverage.
3. User sees which competing offer can be cited honestly.

Expected outcome: the product improves negotiation without fabricating pressure.

## Flow 4: Show Delta

1. User runs or views a follow-up negotiation.
2. System records before and after terms.
3. User sees the verified savings or term improvement.

Expected outcome: the demo proves a measurable result.

## Flow 5: Final Recommendation

1. System ranks vendors by comparable total and risk.
2. Evidence-backed recommendation is shown.
3. User can inspect supporting transcript spans.

Expected outcome: the buyer can choose confidently.
