# Outcome Map

## Core Outcome

Quote Delta Ledger should prove that AI can improve negotiation outcomes while keeping every recommendation tied to evidence.

## User Outcomes

- The buyer understands the true all-in cost of each vendor.
- The buyer can see which fees are confirmed, vague, hidden, contradicted, waived, or declined.
- The buyer knows which competing offer can be cited honestly.
- The buyer gets a final recommendation that can be inspected, not merely trusted.

## Product Outcomes

- Calls become structured quote records.
- Quote records remain linked to transcript evidence.
- Negotiation leverage is filtered through deterministic guardrails.
- The final delta shows price or term improvement after follow-up negotiation.

## Demo Outcomes

- Judges see the confirmed job spec.
- Judges see three comparable vendor offers.
- Judges see evidence attached to quote fields.
- Judges see unsupported leverage blocked.
- Judges see a verified before/after delta.

## Technical Outcomes

- The web MVP stays isolated in `apps/web`.
- API integrations have a clear future boundary in `apps/api`.
- Shared contracts have a place to move when reuse appears.
- The repository can be verified with one command.

## Judging Proof Points

- Innovation: evidence-backed negotiation ledger, not just transcript summarization.
- Usefulness: visible savings or better terms.
- Trust: source-backed quote assertions.
- Execution: working UI, typed logic, tests, and e2e verification.
